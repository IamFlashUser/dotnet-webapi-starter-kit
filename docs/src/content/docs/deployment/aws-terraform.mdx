## Overview

The starter kit includes a complete Infrastructure as Code setup for AWS using Terraform. The infrastructure spans VPC networking, ECS Fargate for container orchestration, Application Load Balancer, RDS PostgreSQL, ElastiCache Redis, and S3 storage. Separate configurations for dev and prod environments allow you to optimize costs during development while maintaining high availability in production.

Each module is self-contained with its own variables, outputs, and resource definitions. The `app_stack` module composes them together, and the `envs` directories provide environment-specific variable values.

## Bootstrap

Before deploying any infrastructure, create the S3 backend for Terraform state:

<Steps>

### Navigate to the bootstrap directory

```bash
cd terraform/bootstrap
```

### Initialize Terraform

```bash
terraform init
```

### Apply the bootstrap configuration

```bash
terraform apply
```

</Steps>

This creates an S3 bucket with encryption and DynamoDB table for state locking. All subsequent Terraform operations store their state remotely in this bucket.

## Deploy

<Steps>

### Navigate to the target environment

```bash
cd terraform/apps/playground/envs/dev/us-east-1
```

### Initialize Terraform with the remote backend

```bash
terraform init
```

### Review the execution plan

```bash
terraform plan
```

### Apply the infrastructure changes

```bash
terraform apply
```

</Steps>

## Dev vs Prod Differences

| Aspect | Dev | Prod |
|--------|-----|------|
| VPC CIDR | `10.10.0.0/16` | `10.30.0.0/16` |
| Subnets | 2 per type | 3 per type |
| NAT Gateway | Single (cost saving) | Per-AZ (HA) |
| RDS | Single-AZ | Multi-AZ, 30-day backups |
| Redis | t3.micro, 1 node | t3.medium, 2 nodes + failover |
| ECS | 1 instance, Fargate Spot | 3 instances, standard Fargate |
| Flow Logs | Disabled | Enabled (90-day retention) |

## VPC Endpoints

The Terraform configuration provisions VPC endpoints for the following AWS services:

- **ECR** (Docker and API) вҖ” Pulls container images without traversing the public internet
- **CloudWatch Logs** вҖ” Sends logs from ECS tasks directly through the private network
- **Secrets Manager** вҖ” Retrieves secrets at runtime without public internet access

These endpoints reduce data transfer costs and improve security by keeping traffic within the AWS network.

<Aside type="warning">

Always review `terraform plan` output before applying. Production changes should go through a PR-based review process with at least one approval.

</Aside>

<Aside type="tip">

The dev environment uses Fargate Spot and a single NAT Gateway to minimize costs. Expect occasional task interruptions with Spot capacity, which ECS handles automatically by rescheduling tasks.

</Aside>
